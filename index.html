Why migrate from Slack to Matrix

How Open Source works at WordPress

  • Why chat is important to the WordPress community
  • Coordination
  • Recurring text meetings at certain times for each meeting
  • You have a time and date where you can bring issues forward and the people working on it will be there
https://make.wordpress.org/meetings/
https://make.wordpress.org/

Things that make Slack hard for an open source community

  • E-Mail Address signup / invites
  • Common issue at Wordcamps where we onboard new people
    • Full page of instructions → people won’t read it
  • Message retention
    • 10,000 / 90 days message history limit for free plans
  • Data is siloed behind’s Slack’s doors

Matrix can do this better

  • Federation is designed so that anyone can join
  • → No retention limits
  • Open Source chat for an Open Source project

Matrix is still behind on

  • Notification Management (big time)
  • Search
  • Access Control for home server members
    • Anyone can create a new room on the server
    • Matrix Corporal exists but we haven’t configured it

Making it easier to join

  • Chatrix
    • Powered by Hydrogen with upstream contributions:
      • Webworkers for Sync to allow multiple Hydrogens open in a single browser → multiple Chatrix
      • more...
  • SSO with WordPress.org
https://make.wordpress.org/polyglots/chat/

Making the Migration easier with a bridge

  • Created the users from Slack with a script (not open source yet)
  • Slack bridge
    • We’re using a forked version of matrix-appservice-slack
      • We submitted some fixes upstream, please merge? :)
    • Modified the bridge to puppet users on the Matrix side since we know the users exist

Migrating history seamlessly

  • Create room with bridge first (no history)
  • Create a second room and migrate the history there
  • Invite everyone who is a member on Slack to the new room
    • (who hasn't said anything yet)
  • Copy events from the first room for catching up to real-time
  • Change room aliases, delete old room

Still missing: App migration

  • Simple apps can be used from Matrix through the Slack bridge
  • Ecosystem problem
    • Companies build Slack apps using internal APIs
    • → Cannot replicate on Matrix
    • Solution might be: bridge from Slack?
